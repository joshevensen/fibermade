# Task 05 — UI: Payment Status, Shopify Link, Tracking, Retail Read-Only

## Starting Prompt

> I'm working through the Shopify orders spec at `specs/shopify-orders/`. Please read `specs/shopify-orders/overview.md` and `specs/shopify-orders/tasks/05-ui.md`, then implement this task. Work through the checklist before marking done.

## Goal

Update the orders index and show pages to handle retail orders correctly: display payment status, surface a "View in Shopify" link, show tracking info, and suppress write actions (edit, transition buttons) for retail orders.

---

## Current State

Read the existing order index and show Vue pages before starting:
- `resources/js/pages/creator/orders/IndexPage.vue` (or similar)
- `resources/js/pages/creator/orders/ShowPage.vue` (or similar)

Understand what props are passed from the controller, what components are used, and what actions are displayed. The goal is to extend these pages minimally — add what's needed for retail, don't restructure what exists.

---

## What to Build

### 1. `OrderResource` / controller props — add new fields

Wherever order data is sent to the frontend (likely an `OrderResource` or inline in the controller), add:

- `payment_status` — string value of the enum (e.g. `"paid"`)
- `type` — already present, confirm it's exposed
- `shopify_admin_url` — pulled from `ExternalIdentifier.data.admin_url` (null for non-retail orders)
- `tracking` — array from `ExternalIdentifier.data.tracking` (null/empty for non-retail)

For the index page, include `payment_status`, `type`, and `shopify_admin_url` on the list resource. For the show page, include all four.

---

### 2. Orders index page

**Payment status badge** — Add a payment status badge column alongside the existing status badge. Use the same badge component/style already in use. Label and color mapping:

| `PaymentStatus` | Label | Color suggestion |
|---|---|---|
| `paid` | Paid | Green |
| `unpaid` | Unpaid | Yellow |
| `partially_refunded` | Partially Refunded | Orange |
| `refunded` | Refunded | Red |

**Retail type indicator** — If `type === 'retail'`, show a "Shopify" badge or label on the row so creators can distinguish retail from wholesale at a glance. Follow existing badge patterns.

**"View in Shopify" link** — If `shopify_admin_url` is present, add a small external link icon/button on the row that opens the Shopify admin URL in a new tab. Keep it subtle — it should not dominate the row.

---

### 3. Orders show page

**Payment status** — Display `payment_status` prominently alongside order status. Use the same badge as the index.

**"View in Shopify" button** — If `shopify_admin_url` is set, show a prominent "View in Shopify" button (external link, opens new tab) in the action area at the top of the page.

**Tracking section** — If `tracking` is non-empty, render a tracking section below order details:

```
Tracking
  UPS — 1Z999AA10123456784
  [Track Package ↗]  (links to tracking URL)
```

Support multiple tracking entries (one row per entry).

**Read-only for retail** — If `type === 'retail'`, hide or disable:
- Edit button / link
- All status transition buttons (Submit, Accept, Fulfill, Deliver, Cancel)
- Any delete action

Replace these with the "View in Shopify" button only. Retail orders are managed entirely in Shopify.

**Line items display** — For line items where `colorway_id` is null, display `title` and `variant_title` as plain text instead of linking to a Colorway/Base page. For matched line items, keep existing behavior.
